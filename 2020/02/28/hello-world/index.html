Windows Powershell 脚本执行策略

发表于 2020-02-28 更新于 2020-02-29 分类于 Web

问题来源

新换笔记本，准备在Github上重建博客，装Git，Node.js, HEXO, 一切正常。在在部署 HEXO 时，PowerShell出现错误。

$ hexo : 无法加载文件 ..\AppData\Roaming\npm\hexo.ps1，因为在此系统上禁止运行脚本。有关详细信息，请参阅 https:/go.microsoft.com/fwlink/?LinkID=135170 中的 about_Execution_Policies。

问题解决

以管理员身份运行 PowerShell

原因分析

首次在计算机上启动 PowerShell 时， 现用执行策略很可能是 Restricted（默认设置）。Restricted策略不允许任何脚本运行。用 get-executionpolicy 查看算机现用执行策略。要在本地计算机上运行未签名脚本（或其它用户的签名脚本），使用命令 set-executionpolicy remotesigned 更改执行策略为 RemoteSigned。

补充说明

PowerShell脚本5种执行权限

  1. Restricted 默认设置，不允许任何脚本运行；
  2. AllSigned 只运行经过数字证书签名的脚本；
  3. RemoteSigned 运行本地的脚本不需要数字签名，但运行从网络上下载的脚本就必须要有数字签名；
  4. Unrestricted 允许所有的脚本运行；
  5. Undefined 在Windows10下这是默认的值，表示未设置任何执行权限，这个值一般是用来删除执行策略。要删除特定范围的执行策略，请将执行策略设置为 Undefined。
