用HEXO在GitHub上搭建个人博客

发表于 2020-02-29 更新于 2020-03-02 分类于 Web

新冠疫情期间，宅在家里，琢磨着学点新东西，做点有意思的事。先是囫囵吞枣翻看了几本HTML5，CSS3和微信小程序开发的书，并尝试着按照书中的范例操弄，不亦乐乎。后来，新购的华为Matebook14到手后，忙着复工的准备，没有那么大段空闲的时间了，再折腾这些就显得有些不务正业了。几年前，也是在假期中曾在Github上搭建过博客，假期结束上班后，那个半吊子的博客就被搁置，不伦不类，不死不活。现在借着玩弄新电脑的兴奋劲，花了两个晚上，捯饬一下，算是给这个博客续了一口气，顺便也算是重温前段时间看过的那些书。这里记录一下，希望以后再想折腾这博客的时候，不至于忘记的一干二净，无从下手。

Git安装

Git 官网 下载适合自己系统的版本。一般情况下，根据提示进行安装，可以按照默认一路NEXT即可。

  • 选择Git默认编辑器。要选择自己比较熟悉，方便上手的编辑器，因为我电脑里装有Visual Studio Code，平常都用它，所以我选择 “User Visual Studio Code as Git’s default editor”。
  • 配置PATH环境。选择 “Use Git from the Windows Command Prompt” 这个选项只向PATH添加最小的Git包，可以在Git Bash和Windows命令提示符中使用Git。
  • 配置Git Bash的终端模拟器。在目前Win10平台中，完全可以选用 “Use Windowns’ default console window” 也就是使用 CMD，尽管有许多限制，没有特殊需求的情况下，在win10系统中用着还好。

验证安装：Win+R，输入cmd，回车，输入 git --version , 查看git版本。若电脑返回的类似 git version 2.25.0.windowns.1 ， OK了。

用HEXO框架在Github上建博客，似乎不需要更多的Git知识了。简单的理解，Git就是HEXO用来上传网页文件到Github的后台小工具。

Github设置

GitHub 官网 注册账号（这里假定注册账号为 yourname)，并建立一个新的 repository, 命名为 yourname.github.io 。要注意这个 repository 名称的第一部分一定要和Github注册的用户名完全一致。如果你注册的账户名为 zhangsan 那么这个 repository 的名称一定要是 zhangsan.github.io 。

这样你的GitHub博客已经建成了，不过这个博客什么都没有，是空白的。下面就是要用HEXO往里面添内容了。

为了保证HEXO能顺利使用Git这个工具往GitHub上传内容，还要设置一下，让你的电脑和GitHub账号建立联系和信任。

  1. 检查电脑里是否已经存在 SSH Key。SSH是啥，就是一种比较安全的网络协议，用于计算机之间的加密登录。SSH Key就是这两个计算机间建立信任的指纹密码。SSH key 分为私钥和公钥，就是两台计算机进行加密、解密、验证用的东西。私钥就是你自己的，在自己手里的钥匙，公钥就是上传给你信任的计算机上，交给别人的钥匙，只有这两个钥匙同时作用，两台计算机才能建立起信任，才好进行沟通。
    ssh key 在win10系统一般存放在 Windows(C:)>用户>YourName>.ssh 这个目录下，私钥的文件名为 id_rsa,公钥的文件名为 id_rsa.pub。如果你在自己电脑上找不到 .ssh 文件夹和这两个文件，则表明你的电脑还没有生成 SSH key。
    另一种办法是在Git Bash中查询。如果上一步Git安装正常的话，那么在win10资源管理器界面或者桌面（一回事）单击鼠标右键，选择 Git Bash Here，在弹出的终端窗口输入 $ ls -al ~/.ssh。如果SSH key存在的话，会显示出文件列表。

  2. 生成SSH Key。打开Git Bash，输入以下命令：ssh-keygen -t rsa -C "youremail@example.com"
    用 -t 对密钥类型指定。如果没有指定则默认生成用于SSH-2的RSA密钥。这里使用的是rsa。同时在密钥中有一个注释字段，用-C来指定所指定的注释，可以方便用户标识这个密钥，指出密钥的用途或其他有用的信息。所以在这里输入自己的邮箱或者其他都行,当然，如果不想要这些可以直接输入：ssh-keygen 。输入完毕后按回车，程序会要求输入一个密码，输入完密码后按回车会要求再确认一次密码，如果不想要密码可以在要求输入密码的时候按两次回次，表示密码为空，并且确认密码为空，此时Windows(C:)>用户>YourName>.ssh 目录下已经生成好了。

  3. 将SSH key添加到GitHub。登录GitHub，打开 setting->SSH keys, 点击右上角 New SSH key, 把Windows(C:)>用户>YourName>.ssh 目录下生成好的公钥 id_rsa.pub 文件以文本打开复制其中全部内容到 key 输入框中，顺便为这个key起个好记的名字，以示区分。

这样GitHub就与你的电脑建立信任，可以一起愉快的玩耍了。

Node.js安装

Node.js 是以Google Chrome V8 引擎为基础的 JavaScript 运行环境，也是HEXO运行的依赖环境。在Node.js 官网 下载对应的版本，依照提示，一路选择默认，Next到底即可。安装完成后，可以在Windows命令行(CMD)窗口查验，输入 node -v 及 npm -v 查验安装的node.js, npm 的版本。

HEXO安装

1. 安装

打开Git Bash终端窗口，输入以下命令： npm install hexo-cli -g

2. 本地部署

在电脑上为此博客建立一个专门的文件夹，比如在D盘根目录下Github-Hexo文件夹D:\Github-Hexo。 使用资源管理器打开此文件夹，在窗口内单击鼠标右键选择 Git Bash Here 打开Git Bash终端窗口。在此窗口输入，依次输入hexo init，hexo generate, hexo server 。 至此本地部署已经完成，启动了hexo server服务，可以在浏览器输入http://localhost:4000 查看HEXO默认搭建的页面效果。

3. Github部署

修改后保存文件。依旧在此文件夹下，打开Git Bash终端窗口，输入npm install hexo-deploy-git-save 安装插件。 插件安装成功后，在此终端窗口分别输入hexo clean, hexo g -d生成并提交至Github发表。顺利的话用浏览器 http://yourname.github.io可以浏览访问博客了，呈现的内容和页面效果和本地部署时浏览器查看时候一样。

博客文章撰写及发布

在博客所在文件下打开git Bash终端窗口，输入: hexo new postname, HEXO会在../source/_posts/目录下生成 postname.md 文件，用markdown编辑器修改并保存此文件（博客文章），然后在git Bash终端输入 hexo g -d,写好的博客文件就被发布到github供浏览访问了（可能会有延时，多刷新几次浏览器窗口，或者清除cookies重新开启浏览器就应该可以看到新撰写的博客文章了。）

主题的配置

每个主题都有不同的特色功能和表现效果，要想充分使用这些功能，最好的办法就是研读你官方文档。对于NexT主题，主要是修改主题配置文件_config.yml(是的，和HEXO的配置文件名是一样的，不过所在文件夹不同，HEXO配置文件在网站文件夹的根目录下，NexT主题的配置文件在D:\Github-Hexo\themes\next\source 文件夹下）

两个配置文件的修改，可以完成许多博客个性化的东西，比如微信、微博等社交图标链接，网站图标，作者头像，字体设置等等。喜欢折腾的的还可以去做各种妖艳的背景和鼠标点击效果，去添加各种访问统计信息显示等。

表现是为内容服务的，如果网站没啥干货，添加许多纯粹的炫酷效果和功能，除了浪费资源，增加网站负担和不稳定性外，也就是自嗨的作用了。
